*All 14 agents reporting to Coda. Phase 4+, queue clear.*

This document is a point-in-time synthesis of every agent's domain audit. It should be treated as a dated snapshot — findings here may have been resolved by the time you read this. Cross-reference open issues and git log before acting on anything listed below.

## Critical — Fix Before New Features

These are actively broken or misleading to users right now.

**Auto-chain and Manual modes do nothing.** `InteractionModeSwitcher` lets users select them, they persist, but `App.tsx:handleSend` always broadcasts in parallel. A user who selects Auto-chain gets parallel behavior silently. Either implement or disable with a "coming soon" tooltip. *(Flint, Atlas, Aria)*

**Ghost mode is completely unwired.** `App.tsx` hard-codes `isGhostMode={false}`. The full infrastructure — `GhostModeManager`, `useGhostMode`, provider guards, tests — is complete and passing. The toggle button and wiring in App are just absent. *(Flint, Aria, Vault)*

**`bg-error` on destructive buttons in `ProviderSettingsPanel.tsx` lines 396 and 431.** The documented decision (in HANDOFF.md) is `NEVER use bg-error text-white — use bg-error-bg`. Sidebar correctly uses `bg-error-bg`. ProviderSettingsPanel doesn't. Contrast failure in some themes. *(Flint)*

**Skip-to-main-content link is absent.** WCAG 2.4.1 failure. Keyboard users must Tab through every sidebar thread row to reach the InputBar. One `<a href="#main-content">` at the top of `AppLayout.tsx` fixes it. *(Ada)*

**`BulkActionBar` confirm-delete has no focus management.** When the confirm state opens, focus stays on the now-gone button. Same issue that was fixed in `ProviderRow` (#115) and `ThreadActionMenu` — `BulkActionBar` was missed. *(Ada)*

**Backend tests never run in CI.** 63 backend tests exist, are fast (~835ms), and all pass locally. The `backend-lint-build` CI job runs lint and tsc only. `npm run test` is one line away. *(Forge, Bastion)*

## Short-Term Wins — Highest ROI (1–2 sessions each)

**Markdown rendering in message bubbles.** The single largest UX gap. Models return markdown; users see raw `**text**` and triple backticks. `MessageBubble.tsx:121` renders plain text. Requires library choice (marked/micromark), XSS sanitization, and streaming-aware rendering. *(Aria)*

**SSE parser extraction — Atlas.** `gpt.ts`, `grok.ts`, `deepseek.ts`, `mistral.ts`, and `generic.ts` each copy ~50 lines of identical SSE loop logic. `mapHttpStatusToErrorCode` and `buildModelError` are copied 5–6 times each. One `openai-sse.ts` helper collapses ~800 lines of duplication. *(Atlas)*

**Fix streaming shimmer for all 6 models.** `index.css` only defines `[data-model="claude"]` and `[data-model="gpt"]` shimmer variants. Gemini, Grok, DeepSeek, and Mistral show the Claude amber shimmer while streaming. Adding 4 variants is ~12 lines of CSS. *(Spark)*

**Wire `useCredentials` and `ApiKeyPanel` to all 6 providers.** Both hardcode `['anthropic', 'openai']`. Four providers (Google, xAI, DeepSeek, Mistral) are invisible to the credential hook. `CREDENTIAL_LABELS` already has all six entries. One-line fix in each file. *(Gate)*

**Wire `filterByArchiveStatus` and `deriveExistingGroups` from `sidebarUtils.ts`.** Both utilities are extracted, tested, and exported — but `Sidebar.tsx` reimplements them inline anyway. *(Aria)*

**Add `backend-test` job to `ci.yml`.** Requires `apt-get install -y python3 make g++` for `better-sqlite3`. `cd backend && npm run test` after lint passes. Runs parallel with frontend test, adds ~2s wall time. *(Forge, Bastion)*

**Update token schema and tailwind-mapping for wave-2 accents.** `schema.md` lists 4 accent keys; all 7 theme files carry 7. `tailwind-mapping.md` is missing `accent-grok`, `accent-deepseek`, `accent-mistral`. Gate's future custom-theme validator will reject valid themes until this is fixed. *(Luma)*

**Fix `ProviderSettingsPanel` width to track sidebar width.** Hardcoded `calc(100vw - 256px)` at line 919. The sidebar is drag-resizable (200–480px). On narrow sidebar configs the panel overlaps. `sidebarWidth` is already tracked in state. *(Aria)*

**Add the 7 skipped `ThreadActionMenu`/`BulkActionBar` tests.** `Sidebar.management.test.ts` skips them with a comment saying `@testing-library/react` isn't available — but RTL is already in devDependencies and used in 6 other files. *(Scout)*

**Add Forge and Bastion to both CONTRIBUTING.md tables.** Both agents are live and have committed code. Neither appears in the agent boundary table or agent profiles table. *(Quill)*

**Fix `0m` relative timestamp.** `Sidebar.tsx:103` shows `0m` for conversations updated in the last minute. One condition for `diffMinutes === 0` returning `'just now'` fixes it. *(Aria)*

**Add Escape key to clear directed-reply pill.** `InputBar.tsx:119` handles Enter only. One `else if (e.key === 'Escape' && directedReplyTarget)` line. *(Aria)*

**Widen `SendMessageFn` to match implementation.** `SendMessageOptions` in the types file doesn't include `conversation` or `systemPrompt`, but the actual `sendMessage` requires them. The public contract is narrower than the implementation. *(Arch)*

**Define `SessionTokenUsage` as intersection.** `type SessionTokenUsage = TokenUsage & { modelId: ModelId }` — eliminates field duplication; any future `TokenUsage` field propagates automatically. *(Arch)*

**`providerRoster.ts` unit tests.** Gate's most complex file — roster CRUD, ID collision resolution, corrupt-JSON resilience — has zero unit tests. Only indirect coverage through a heavily mocked panel test. ~20 tests. *(Scout)*

## Architecture & DRY

**`Sidebar.tsx` (1499 lines) and `ModelSelectorPanel.tsx` (1215 lines) need splitting.** Both contain 6–10 discrete sub-components. `ThreadActionMenu` alone is 295 lines of keyboard navigation logic. *(Aria)*

**No shared icon system.** ~25 SVG icons are inlined in the files that use them. The close/× path appears in 4 files; the gear appears in 3 slightly different sizes. One `/src/ui/icons.tsx` eliminates this. *(Aria)*

**`getModelDotStyle` defined 3 independent times.** `Sidebar.tsx:147`, `ModelSelectorPanel.tsx:40`, and `ProviderSettingsPanel.tsx:43`. *(Aria)*

**`useClickOutside` pattern implemented 4 different ways.** `AddModelButton`, `ExportButton`, `AccentColorPicker`, and `ThreadActionMenu` each roll their own outside-click closing with three different event types. One hook. *(Aria)*

**`ChevronIcon` duplicated in `ModelSelectorPanel.tsx` and `Sidebar.tsx`.** Identical SVG, identical rotation logic. *(Aria)*

**`BuiltInModelId` re-enumerated in 4 places.** The type in `/src/types/index.ts`, plus `BUILTIN_MODEL_IDS` in `providerRoster.ts`, `VALID_MODEL_IDS` in `modelVersion.ts` and `accentColors.ts`, and `ALL_BUILTIN_IDS` in `ProviderSettingsPanel.tsx`. Adding a 7th provider requires 4+ file updates. *(Arch, Gate)*

**`applyUserAccentColors` has parallel accent maps in `theme.ts` and `AccentColorPicker.tsx`.** Both manually map `modelId → CSS var name`. Adding a provider requires updating both. *(Flint)*

**`MAX_TOKENS = 8096` defined 6 times.** One per model file. Wrong for several providers (Gemini 2.5 Flash supports 65536 output). Needs to live in the registry entry. *(Atlas)*

**`migrateLocalToServer` accepts concrete classes, not `StorageProvider` interface.** The function only calls interface methods. Widening the signature is a one-line change. *(Vault)*

**`GhostModeManager._ghostIds` Set is redundant.** `this._store.has(id)` already answers `isGhost(id)`. The parallel Set is an extra write site to keep consistent. *(Vault)*

**localStorage key naming uses 4 different conventions** across Gate (`rt_key_`, `roundtable:`, `roundtable_`, `rt-ui-`). The three pre-standard keys need a migration-on-read. *(Gate, Arch)*

**Export serializers (`conversationToMarkdown`, `conversationToHtml`) live inside `LocalStorageProvider.ts`** but have no dependency on the provider. Extract to `/src/storage/conversationExport.ts`. *(Vault)*

**`App.tsx` is becoming a god component.** Streaming state management, conversation store integration, roster subscription, version selection, directed reply state, and token computation all live there. The streaming chunk handler (lines 203–255) warrants extraction into a `useStreamingMessages` hook. *(Flint)*

## Long-Term Investments

**`BaseOpenAIProvider` class.** The four OpenAI-compatible built-ins differ only in URL, credential key, and default model. They could each be a config object passed to a base class. `GenericOpenAIProvider` already models this pattern. *(Atlas)*

**React Context or Zustand for cross-cutting state.** `AppLayoutProps` has 30 props, most pass-throughs. `MessageThread`, `Sidebar`, and `ModelSelectorPanel` could read what they need without explicit threading. *(Aria)*

**`StorageProvider` pagination.** `listConversations()` returns all conversations in one call. At hundreds of conversations this is a full scan on every render. The interface needs `cursor?` / `limit?` before the backend scales. Arch decision required. *(Vault)*

**Schema versioning on stored `Conversation` objects.** `parseConversation` casts raw JSON with no version field. A required non-optional type field addition produces broken objects that pass the null check. A `schemaVersion` field + migration function prevents this. *(Vault)*

**Remote/live-API model catalog.** `ModelCatalogSource: 'remote' | 'live-api'` is typed and specced but not implemented. Atlas owns the fetch logic; Aria renders via the same version picker. *(Atlas)*

**Outrun theme full personality treatment.** The entry flash (`#FF00AA` full-viewport overlay, 100ms fade) is fully specced in `motion.md` but absent from the codebase. *(Spark)*

**Chunk fade-in wiring.** `.chunk-entering` CSS exists and is correct, but `MessageBubble.tsx` renders `message.content` as a single text node with no chunk boundaries. Either Atlas exposes chunk boundaries or Aria diffs content between renders. *(Spark, Atlas)*

**Live browser keyboard audit pass.** Focus traps in `AccentColorPicker` and `ProviderSettingsPanel` have `it.fails()` wrappers removed, but the traps have never been verified in a real browser. jsdom doesn't execute real Tab traversal. *(Ada)*

**WCAG 2.1 → 2.2 upgrade path.** 2.5.8 target size (18×18px palette icon is below 24×24px minimum) and 2.4.11 focus-not-obscured are the highest-priority 2.2 criteria to plan for. *(Ada)*

**Release workflow (`release.yml`).** No automated path from a version tag to a Docker image or GitHub Release. Tag → lint/build/test → `ghcr.io` publish → GitHub Release with changelog. *(Forge)*

**`ConversationContext` as narrower Atlas input type.** Atlas only reads `id`, `messages`, and `models` from `Conversation`. A purpose-built type prevents Atlas from accidentally reading Vault-internal fields (`isGhost`, `archivedAt`, `groupId`). *(Arch)*

**Unified error base type.** `ModelError`, `StorageError`, and `BackendAuthError` share no common ancestor in the types file. A future unified error boundary in Aria will need to catch all three. An `AppError` interface or promoted error codes would enable this. *(Arch)*

## Process & Infrastructure

**Playwright E2E is not run in CI.** `smoke.spec.ts` and `phase4.spec.ts` exist and run locally. Zero CI coverage. *(Forge)*

**Coverage report not generated in CI.** No `--coverage` flag on the Vitest run. No threshold step. Coverage regressions merge silently. *(Scout, Forge)*

**Shared backend test fixtures are duplicated.** `insertUser`, `loginAs`, and `makeConversation` are copied verbatim across all 3 backend route test files. A `/backend/tests/helpers/fixtures.ts` eliminates this. *(Bastion)*

**`seedAdminUser()` has zero test coverage.** Called at every backend startup. No test covers the no-op behavior, default-password path, or that the seeded user can log in. *(Bastion)*

**`App.tsx` chunk handler is untested.** The most complex imperative logic in the codebase — accumulates streaming chunks, guards against stale conversation ID, calls `store.updateConversation` — has no integration test covering the full `App → sendMessage → chunk → store` path. *(Flint)*

## Design System

**`BRIEF.md` accent table is stale.** Outrun `model-other` is `#FF00AA` in BRIEF; live theme has `#FF8D77`. Slate `model-gemini` was WCAG-adjusted. Either update the table or add a superseding note. *(Luma)*

**Z-index scale is undocumented.** Provider panel z-40, model selector z-50, color picker z-60 are scattered in one spec file with no canonical scale. `/_design/specs/layers.md` would prevent future overlay collisions. *(Luma)*

**No standalone tooltip spec.** Tooltip appearance is described in-context in two different specs, never extracted as a reusable component spec. *(Luma)*

**No markdown rendering token layer.** `components.md` says "Aria owns markdown styling" but code block backgrounds, inline code, link colors, and list treatment have no token-level specs. In themed environments, hardcoded prose colors won't update with the theme. *(Luma)*

**`favicon.ico` is missing.** `index.html` references it; the file doesn't exist. All 7 PNG icon export sizes (16–512px) are specced but not produced. No PWA manifest. *(Marque)*
